怎样才能称为一个合格的程序员

程序员作为一种职业，其能力主要体现在利用或制造工具解决问题。

程序员应该拥有以下能力：

  1. 基本科学素养
  2. 了解计算机运行原理
  3. 了解代码执行原理
  4. 抽象问题的能力
  5. 学习和快速学习的能力
  6. 通过简单的说明便能掌握工具的能力
  7. 通过经验使用工具的能力
  8. 利用工具抽象问题的能力
  9. 创造8需要的工具的能力
  10. 保证大部分人也能使用9所创造的工具的能力

除了2 3 两条以外，其他能力与计算机领域无关

git常用命令集合

git help： 查看所有git命令与解释

git clone+远程仓库地址 ：克隆远程仓库到本地

git branch+分支名：新建一个分支

git checkout+分支名：切换到某个分支

git pull+远程仓库名+远程分支名：从远程仓库 的分支里，将代码更新到本地当前分支

git status：查看工作状态

git add：添加修改到暂存区

git commit ：将暂存区的内容提交到本地仓库

git log：查看本地参考当前分支的历史提交记录

git push+远程仓库名+远程分支名：将本地仓库更新到远处

微服务架构设计大纲

花草秀是一个大型的、多应用的、多技术栈的、多团队的项目，为了满足花草秀项目发展需要，现提出微服务架构体系。

微服务架构的特点：一个完整的产品可以由多个服务构成，这些服务可以是整个项目的公用组件，场景如下：

产品校园镖局，拥有用户系统，订单系统和支付系统

微服务架构模式下可以拆分为：

用户服务、订单服务、支付服务、

这三个服务并不仅仅是用于校园镖局项目，同时可以用于其他项目，即一个服务并不是针对某一具体产品开发，而是针对某一具体问题，使用一个服务的方式是从服务系统里产生一个实例，这个实例提供了消费方相关的业务逻辑。

设计目标

服务实例给消费者提供配置和数据查询和事件监听功能，服务自身拥有与特定问题相关的逻辑代码，服务拥有实例管理的能力。

服务实例和消费者实例之间使用网络进行通信，服务实例内部使用异步消息队列向消费者发送事件\数据消息。

服务实例应该保证链接的安全性。

服务能够独立部署。

服务可用由任何语言、框架编写。

参加了人生第一次笔试和面试，记录一下心情

2016年6月20日，世界滑板日前夕，我参加了思应科技的招聘，第一轮是笔试，笔试在一台树莓派机器上进行，是在线编程，编程平台叫codility

面试题有3道，两道算法，一道SQL，第一道算法，很快就斩获, 第二道 在一个曲线里面找出最深的开口朝上的二次函数，本来很简单，但是搞错了连续增加的判断方式，导致浪费了很多时间，SQL不会做。

反思

必须多刷点题啊，不然大公司是进不了的

后来看笔试结果的时候我都快哭了，烂的一逼，旁边应聘iOS那哥子已经约好和BOSS远程面试，我已经做好走人的准备。

反转

结果HR叫我准备面试，说BOSS要面我。

然后就是和BOSS愉快的聊了聊天，然后就通过了。

最后
